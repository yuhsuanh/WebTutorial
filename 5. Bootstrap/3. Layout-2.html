Over grid size

If more than 12 columns are placed within a single row, each group of extra columns will, as one unit, wrap onto a new line.

.col-9
.col-4
Since 9 + 4 = 13 > 12, this 4-column-wide div gets wrapped onto a new line as one contiguous unit.
.col-6
Subsequent columns continue along the new line.



Order Classes

Use .order- classes for controlling the visual order of your content.

First in DOM, no order
Second in DOM, order-12
Third in DOM, order-1

There are also responsive .order-first and .order-last classes that change the order of an element by applying order: -1 and order: 13 (order: $columns + 1), respectively

First in DOM, no order
Second in DOM, order-12
Third in DOM, order-1



Offsetting columns

You can offset grid columns in two ways: our responsive .offset- grid classes and our margin utilities.

col-sm-4
offset-sm-4 col-sm-4
.col-sm-3 .offset-sm-3
.col-sm-3 .offset-sm-3
.col-sm-6 .offset-sm-3

Margin utilities

With the move to flexbox in v4, you can use margin utilities like .mr-auto to force sibling columns away from one another.

.col-md-4
.col-md-4 .ml-auto

Nesting

To nest your content with the default grid, add a new .row and set of .col-sm-* columns within an existing.

Level 1: .col-sm-9
Level 2: .col-8 .col-sm-6 (inside level1)
Level 2: .col-4 .col-sm-6 (inside level1)
